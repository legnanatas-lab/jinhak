# 가온길 에듀 GitHub Pages 홈페이지 v4

정적 호스팅(GitHub Pages)에 바로 업로드할 수 있는 가온길 에듀 홈페이지입니다. v4에서는 프로그램 카드와 상세 페이지에 실험·AI·메이커 주제별 실사형 이미지를 적용하고, 첫 화면 배너 이미지도 실제 교육 현장 느낌으로 교체했습니다.

## 포함 기능

- 메인 배너 슬라이드 3종: 과학실험, AI 융합교육, 메이커 교실
- 프로그램별 주제 이미지 적용
  - MBL 과학실험
  - 과학심화교육
  - 생명과학 실험
  - STEAM 융합 프로젝트
  - AI 융합교육
  - MAKER 교실
  - 과학 진로직업 체험
  - 환경 과학 교실
- 공지 팝업 설정
- 게시판 관리
- 문의 폼 및 문의 내역 관리
- 홈 하단 관리자 로그인
- 관리자 페이지에서 로고, 메뉴, 배너, 문구, 게시판, 프로그램 전체 수정
- 관리자 페이지에서 GitHub `assets/data/site-data.json` 직접 업데이트 지원

## 관리자 로그인

- 주소: `admin.html`
- 아이디: `admin`
- 비밀번호: `000000`

홈페이지 제일 아래 푸터에서도 관리자 로그인이 가능합니다.

## GitHub Pages 배포 방법

1. ZIP 파일 압축을 풉니다.
2. 압축을 푼 폴더 안의 파일과 폴더 전체를 GitHub 저장소 루트에 업로드합니다.
3. GitHub 저장소 `Settings → Pages`로 이동합니다.
4. `Deploy from a branch`를 선택합니다.
5. 브랜치를 `main`, 폴더를 `/root`로 설정합니다.
6. 배포 주소가 생성될 때까지 잠시 기다립니다.

## 전체 방문자에게 관리자 수정 내용 반영하기

GitHub Pages는 정적 호스팅이므로, 관리자에서 수정한 내용은 먼저 현재 브라우저 저장소에 저장됩니다. 전체 방문자에게 반영하려면 관리자 페이지의 **데이터 → GitHub에 바로 반영** 기능을 사용하세요.

필요한 정보:

- GitHub 소유자/아이디
- 저장소 이름
- 브랜치: 보통 `main`
- 데이터 파일 경로: `assets/data/site-data.json`
- GitHub Personal Access Token: 해당 저장소의 `Contents: Read and write` 권한 필요

## v4 최적화 및 점검 내역

- PNG 원본 이미지를 WebP로 변환하여 용량 최적화
- 카드 이미지를 16:10 비율로 통일
- 메인 배너 이미지를 1920×900 WebP로 최적화
- 프로그램/갤러리 이미지는 lazy loading 적용
- JavaScript 문법 검사 완료
- JSON 파싱 검사 완료
- 내부 이미지 경로 존재 여부 검사 완료
- ZIP 무결성 검사 완료

## 주의

생성형 이미지 기반 실사형 사진이므로 실제 특정 학교나 학생의 촬영 사진은 아닙니다. 실제 수업 사진이 준비되면 관리자 페이지에서 각 프로그램 이미지를 교체할 수 있습니다.
