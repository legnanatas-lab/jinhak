# 가온길 에듀-가온길 입시 전략 연구소 - GitHub Pages 배포본

이 폴더 안의 파일을 GitHub 저장소 루트에 올리고 GitHub Pages를 켜면 됩니다.

## 올릴 파일

- `index.html`
- `404.html`
- `.nojekyll`
- `assets/styles.css`
- `assets/config.js`
- `assets/app.js`
- `supabase-schema.sql`
- `supabase/functions/gijang-api/index.ts`

## GitHub Pages 설정

1. GitHub 저장소를 만듭니다.
2. 이 폴더 안의 파일 전체를 저장소 루트에 업로드합니다.
3. GitHub 저장소의 `Settings > Pages`로 이동합니다.
4. `Build and deployment > Source`를 `Deploy from a branch`로 설정합니다.
5. Branch는 `main`, 폴더는 `/root`를 선택합니다.
6. 저장 후 표시되는 GitHub Pages 주소로 접속합니다.

## Supabase DB 연결

1. [Supabase](https://supabase.com)에서 새 프로젝트를 만듭니다.
2. Supabase 좌측 `SQL Editor`에서 `supabase-schema.sql` 내용을 전체 실행합니다.
3. Supabase `Authentication > Users`에서 관리자 사용자를 만듭니다.
   - Email: `admin@gijang.local`
   - Password: `0000`
   - Email Confirm: 체크
4. 생성된 admin 사용자의 UUID를 복사해 `supabase-schema.sql` 하단 안내 INSERT의 `관리자_AUTH_USER_UUID` 자리에 넣고 실행합니다.
5. Supabase CLI가 설치된 터미널에서 이 폴더 기준으로 Edge Function을 배포합니다.

```bash
supabase functions deploy gijang-api --no-verify-jwt
```

6. Supabase `Project Settings > API Keys`에서 아래 두 값을 확인합니다.
   - Project URL
   - publishable key
7. 이 배포본의 `assets/config.js`에는 현재 만든 `Career Center` 프로젝트 값이 이미 입력되어 있습니다. 다른 Supabase 프로젝트로 바꿀 때만 아래 값을 수정합니다.

```js
window.GIJANG_DB_CONFIG = {
  supabase: {
    url: "https://프로젝트아이디.supabase.co",
    anonKey: "여기에-publishable-key",
    functionName: "gijang-api",
    authEmailDomain: "gijang.local"
  }
};
```

8. `assets/config.js`를 바꾼 경우 GitHub에 다시 업로드/커밋합니다.

> `__MACOSX` 폴더나 `._`로 시작하는 파일은 macOS 압축 메타데이터이므로 GitHub에 올리지 않아도 됩니다.

## 로그인

- 관리자: `admin / 0000`
- 컨설턴트 예시: `andong3 / 000000`

Supabase Auth API로 생성되는 컨설턴트 계정은 비밀번호 6자 이상이 필요합니다.

## 포함된 기능

- 메뉴/하위 메뉴 보기
- 프로그램 참가신청
- 같은 컨설턴트, 같은 날짜, 같은 시간 중복 신청 방지
- 로그인 후 신청확인
- admin 전체 신청 확인 및 엑셀 다운로드
- 메인화면 우측 달력/센터 일정은 관리자 페이지 `프로그램/일정`에 등록된 신청 가능 일정 기준으로 표시
- 상담 신청 탭에서 날짜별 컨설턴트 배정 시간, 예약자 현황 함께 확인
- 관리자 상담 신청 탭에서 예약자 정보 확인/수정, 예약취소, 예약 이동, 날짜·시간·컨설턴트 변경, 삭제 가능
- 컨설턴트 본인 신청 및 본인에게 배정된 일정 확인
- 관리자 페이지에서 메뉴, 본문, 배너, 로고, 팝업, 게시판, 계정, 비밀번호 수정
- 관리자 페이지에서 프로그램별 신청 가능 날짜, 시간, 장소, 담당 선생님 수정
- 관리자 페이지에서 컨설턴트별 프로그램 일정관리 권한 부여

## 이번 수정본에서 바뀐 점

- 기존 메인화면에 남아 있던 고정 센터 일정 표시를 사용하지 않고, 관리자 페이지의 `프로그램/일정` 데이터만 달력과 센터 일정에 반영합니다.
- 관리자 페이지 `상담 신청`에서 예약자를 취소 상태로 바꾸거나, 열어 둔 다른 일정/시간으로 이동할 수 있습니다.
- 예약자의 이름, 신청자 구분, 거주지, 학교, 학년, 학부모/학생 연락처, 상담 희망 내용, 관리자 메모가 함께 표시되고 수정됩니다.
- Supabase를 사용하는 경우 이 기능은 Edge Function 재배포 후 적용됩니다.

## 꼭 알아둘 점

Supabase 설정값이 비어 있으면 브라우저 `localStorage`에만 저장됩니다.
Supabase 설정값과 Edge Function을 연결하면 신청자 정보와 관리자 수정 내용이 Supabase DB에 저장되어 여러 컴퓨터에서 공유됩니다.

보안 모드에서는 브라우저가 DB 테이블을 직접 수정하지 않습니다.
관리자/컨설턴트 권한 확인과 DB 쓰기는 `gijang-api` Edge Function에서 처리합니다.

- 첫 화면 전체 로딩창은 사용하지 않고, 필요한 경우 메인 배너 영역에만 밝은 대기 화면을 표시합니다.
